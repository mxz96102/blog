Chapter 1.1

What is Object Orientation?

  • Object Orientation is a kind of thinking methodology.
    是一种思维方式，是观察和分析问题的方法或观点。

    Object-Oriented approach:
    思考的著眼点在系统所需用到的对象以及对象之间的互动。以对象（类）为单位

  • Object-Orientation is a kind of design methodology
    是一种程序设计模式。

    Objects are the building blocks of the program
    对象是程序中的基础元素

    Objects represent real-world abstractions within the application
    应用程序中的对象代表客观事物的抽象

  • OOP是一种计算机编程架构，精髓在于对程序
    的组织与构造。

    OOP将对象作为程序的基本结构单元，程序由
    对象组合而成。

面向对象的基本观点

  • 客观世界由对象组成，任何客观实体都是对象 ，复杂对象可以由简单对象组成
  • 具有相同属性和操作的对象可归纳成类，对象是类的一个实例。
  • 对象之间的联系通过消息传递来维系。

面向对象的应用程序（Application）主要特点

  • Application由类的定义和类的使用两部分组成, 在主程序（主动类）中创建各类的对象并规定它们之间传递消息的顺序。
  • 对象中的操作被执行都是通过向该对象发送消息来实现的。 该对象接收到消息后, 启动相应方法完成相关操作。

面向对象优点

面向对象程序设计优于传统的结构化程序设计

  • 它有希望解决软件工程的两个主要的问题：软件复杂性控制和软件生产率的提高。
  • 达到软件工程的三个主要目标：重用性,灵活性,扩展性。

Object Oriented Analysis 面向对象分析

  • OOA是指利用面向对象的概念和方法为软件需求建造模型，使用户的需求逐步精确化、一致化、完全化的分析过程（建立问题域模型）。
    • 首先应该明确用户的需求
    • 然后再将这些需求以标准化模型的形式规范地表述出来

Object Oriented Design 面向对象设计

  • OOD是把分析阶段得到的需求转变成系统实现方案的过程（建立求解域模型）。
    • 设计的目的是指定一个可行的解决方案，构建类图，以便能很容易地将需求转变成为编程代码。
    • 在设计阶段，并不牵涉某一种具体的计算机语言，而是用一种更通用的描述工具。

Chapter 1.2

Object, Attributes and Methods

  • Object
    • It is a thing （实体，事物）that has attributes and
      behaviors.
    • object identity
  • Attributes （对象的静态特征，一般用数据表示）
    • describe the properties of an object.
    • Such as color, weight, name,…
  • Behaviors (对象的动态特征，用methods, or operations描述 )
    • describe what an object can do（对象的行为是描述这个对象能做什么）
    • so using methods define the behavior of an object.（一般用方法（methods ）表示之）

对象

  • 对象是系统中用来描述客观事物的一个实体，它是构成系统的一个基本单位。
  • 对象可以是具体的、有形的物(如人、汽车，订单），也可以是无形的事物或概念
  • 对象是一个封装了属性和行为的实体
  • 对象只描述客观事物本质的、与系统目标有关的特征，而不考虑那些非本质的、与系统目标无关的特征。

Main Types of Objects in software system

  • Graphic User Interface (GUI) objects
  • Problem domain objects
  • Data access object

Objects Interaction and Messages

  • In an application, the objects interact and communicate with each other to complete the tasks.
  • Objects interact by sending messages to each other, asking another object to invoke one of its methods.
    • 消息：是向对象发出的服务请求。服务通常被称为方法（method）或函数（function）。对象之间通过消息进行交互。
  • 对象之间的消息传递机制对应于面向过程程序设计的过程调用。
    • 消息传递的实质就是方法的调用。只不过方法的调用受到消息的控制, 而过程调用是直接的。
  • 消息与方法的关系：
    • 消息的实现是通过对象中所定义的方法(method)来完成
    • 对象根据接收到的消息，调用相应的方法；有方法，对象才能响应相应的消息。

类和实例

  • Class （类）
    • Is a set of objects with same attributes and methods.
  • Instance （实例）
    • Objects can be referred to as instances（实例） of the class ：对象可以被称为类的实例（实例）
    • When an object is created for the class, it is common to say the class is instantiated
    • The terms “instance” and “object” are often used interchangeably

Encapsulation & Information Hiding （封装与信息隐藏）

  • Encapsulation（封装）
    • 是指把对象的属性和操作封装进一个单元，它的内部信息对外界隐藏，不允许外界直接存取对象的属性，只能通过对象提供的有限接口对对象属性数据进行操作。
    • 对于外界来说，只能知晓对象的外部行为而无法了解对象行为的内部细节，这个可以保证对象内部属性数据的安全性。
    • Encapsulation是面向对象方法的重要原则，也是类的一个特性。
  • Encapsulation特点：
    • 保护数据成员（属性），不让该类以外的程序直接访问或修改，只能通过提供的公共的接口（方法名及参数）访问
    • 方法的细节对外部是隐藏的，只要接口不变，内容的修改不会影响到外部的调用者
    • 从对象外面不能直接访问对象的属性，只能通过和该属性对应的方法访问。（get、set）
    • 对象的方法可以接收对象外面的消息。
  • Information hiding
    • Using encapsulation to hide the internal structure of objects, protecting them from corruption
  • 封装的作用
    • 对象的数据封装特性彻底消除了传统结构方法中数据与操作分离所带来的种种问题，提高了程序的可复用性和可维护性，降低了程序员保持数据与操作相容的负担。
    • 对象的数据封装特性还可以把对象的私有数据和公共数据分离开，保护了私有数据，减少了可能的模块间干扰，达到降低程序复杂性、提高可控性的目的。

Inheritance （继承）

  • one class of objects takeson characteristics of another class andextends them. （继承是在已有类（superclass）的基础上派生出新的类（subclass），新的类能够继承已有类的属性和行为，并扩展新的能力。）
  • 继承，也称泛化，它是面向对象描述类之间相似性的一个重要机制。
  • 继承是指子类可以自动拥有父类的全部属性与方法。
  • 继承的作用：
    • 软件重用。避免公用代码的重复开发, 减少代码和数据冗余;
    • 便于扩展或改进。

Polymorphism （多态）

  • Is a term that describes a situation where one method’s name may refer to different methods. (不同的方法可以有相同的方法名)

Benefits of OO concepts

  • Class – Remove redundant description
  • Encapsulation – Easy to modify a data structure of an object without infecting others. 维护变得简单
  • Inheritance – Reuse class and Easy to extension of functions by subclass
  • Polymorphism – Reduce the complexity of a program; easy to change（override） object’s method

面向对象方法的主要特点

  • 从问题域中存在的客观事物，用对象作为这些事物的抽象表示，并以此作为软件系统的基本构成单位。
  • 事物的静态特征用对象的属性表示，事物的动态特征用对象的服务（方法）表示。
  • 对象的属性与服务结合为一体，成为一个独立的实体，对外屏蔽其内部细节（称为封装）。
  • 对事物进行分类。具有相同属性/服务的对象归为一类，类是这些对象的抽象描述，每个对象是类的一个实例。

面向对象开发方法的优点

  • 与人们习惯的思维方法一致
    • 把软件系统看成是各种对象的集合，这更接近于人类的自然思维方式。
  • 可使软件系统结构更加稳定
    • 因为封装－－对象是最小构造块
  • 支持封装，抽象，继承和多态，提高了软件的可重用性，可维护性和可扩展性

OOP设计的一般方法和技巧

  • 面向对象程序设计方法提供了一种新的系统设计模型，就是将软件系统看成是对象的集合，而对象是通过交互作用来完成任务的，每个对象用自己的方法管理数据。
  • 一个好的程序设计应具备的特征：
    • 良好的可读性
    • 可重用性
    • 可维护性
    • 可扩展性
